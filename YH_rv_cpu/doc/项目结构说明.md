# 项目结构说明

## 目标

这份说明用于明确 `icdc_workspace` 中 `YH_rv_cpu` 的目录边界，避免源码、文档、构建结果和运行时临时文件混放。

## 正式目录

- `rtl/`
  - CPU 核心、SoC 包装和存储器包装等正式 RTL 设计文件
- `tb/`
  - testbench、专项 smoke test 和回归入口
- `sw/`
  - 固件源码、启动代码、链接脚本、CoreMark 适配、`riscv-tests` 适配
  - 正式验证输入也包含这里的 linker 文件，例如 `sw/linker/YH_rv_cpu_riscv_tests_large.ld`
- `fpga/`
  - FPGA 顶层、约束、Vivado Tcl 脚本和板级相关文件
- `scripts/`
  - 可复用的构建、验证、清理和环境检查脚本
  - 正式验证 manifest 也放在这里，例如：
    - `riscv_tests_rv32_baseline.txt`
    - `riscv_tests_rv64_baseline.txt`
    - `riscv_tests_rv32_ui_all.txt`
    - `riscv_tests_rv64_ui_all.txt`
- `doc/`
  - 技术文档、变更记录、handoff 和任务说明
- `build/`
  - 构建产物、回归摘要、HEX/ELF、报告等需要复查的结果

## 仓库根临时目录

- `../_tmp/`
  - 统一收纳整个 `icdc_workspace` 的 Vivado / xsim / GUI 临时文件
  - 仿真运行时默认落到 `../_tmp/sim_runtime/`
  - 工具日志默认落到 `../_tmp/tool_logs/`

## 根目录收纳规则

`YH_rv_cpu` 根目录默认只保留：

- 正式目录
- `README.md`

以下内容不要长期留在 `YH_rv_cpu` 根目录：

- `xsim.dir`
- `xvlog.log`
- `xelab.log`
- `xsim.log`
- `xsim_*.backup.*`
- `dfx_runtime.txt`
- `vivado.log`
- `vivado.jou`
- `clockInfo.txt`
- 其他一次性缓存与运行时垃圾

## 当前执行约定

- 仿真脚本运行后，应把临时运行目录收纳到 `../_tmp/sim_runtime/<task>/`
- 工具日志统一收纳到 `../_tmp/tool_logs/`
- 手动打开 GUI 时，优先使用 `scripts/open_vivado_project.bat`
- `build/` 下的 summary、report、bitstream 只有在它们承担正式证据职责时才保留，否则应清理或重新归档
- full-ui manifest 和 large linker 现在属于正式输入，不应被当作一次性调试垃圾处理
