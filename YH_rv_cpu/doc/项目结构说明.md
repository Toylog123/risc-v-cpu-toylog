# 项目结构说明

## 目标

这份说明用于明确 `icdc_workspace` 下 `YH_rv_cpu` 的目录边界，避免源码、文档、构建结果和运行时临时文件混放。

## 正式目录

- `rtl/`
  - CPU 核心、SoC 包装和存储器包装等正式 RTL 设计文件。
- `tb/`
  - testbench、专项 smoke test 和回归入口。
- `sw/`
  - 固件源码、启动代码、链接脚本和 CoreMark / riscv-tests 适配内容。
- `fpga/`
  - FPGA 顶层、约束、Vivado Tcl 脚本和板级相关文件。
- `scripts/`
  - 可复用的构建、验证、清理和环境检查脚本。
- `doc/`
  - 技术文档、变更记录、handoff 和任务说明。
- `build/`
  - 保留构建产物、回归摘要、HEX/ELF/DCP/报告等需要复查的结果。

## 仓库根临时目录

- `../_tmp/`
  - 统一收纳整个 `icdc_workspace` 的 Vivado / xsim / GUI 临时文件。
  - 仿真工具运行中生成的 `xsim.dir` 默认归档到 `../_tmp/sim_runtime/`。
  - `xvlog/xelab/xsim` 日志和 `vivado.log/.jou` 默认归档到 `../_tmp/tool_logs/`。
  - Vivado 的本地用户态缓存和 `Temp` 目录也收纳到这里。

## 根目录收纳规则

`YH_rv_cpu` 目录默认只保留：

- 正式目录
- `README.md`

以下内容不要长期留在 `YH_rv_cpu` 根目录：

- `xsim.dir`
- `xvlog.log`
- `xelab.log`
- `xsim.log`
- `xsim_*.backup.*`
- `dfx_runtime.txt`
- `vivado.log`
- `vivado.jou`
- `clockInfo.txt`
- 其他一次性运行时缓存

## 当前执行约定

- 仿真脚本跑完后，会自动把 `YH_rv_cpu` 根目录运行时文件收纳到 `../_tmp/sim_runtime/<任务名>/` 和 `../_tmp/tool_logs/xsim/<任务名>/`
- Vivado 本地用户态目录会落在 `../_tmp/vivado_user/`
- Vivado 批处理日志会落在 `../_tmp/tool_logs/vivado/`
- `clockInfo.txt` 这类时钟调试文件也会统一收纳到 `../_tmp/tool_logs/vivado/clock_debug/`
- 手动打开 GUI 时，优先使用 `scripts/open_vivado_project.bat`，这样 `vivado.log/.jou` 会集中落在 `../_tmp/tool_logs/vivado/`，`.Xil` 也会留在仓库根 `project/`，不会污染源码目录
- 历史遗留的旧工作区和旧临时目录会迁移到 `../_tmp/legacy/`
