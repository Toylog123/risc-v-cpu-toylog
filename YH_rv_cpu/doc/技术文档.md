# YH_rv_cpu 技术文档

## 1. 文档范围

本文档是 `YH_rv_cpu` 当前比赛提交主线的技术基线说明。目标是提供一份和当前脚本、RTL、TB、报告、交付材料一致的单一入口，并明确区分：

- 冻结基线
- 活跃工作区中的扩展验证

当前基线日期：`2026-04-07`
当前活跃验证日期：`2026-04-08`

## 2. 工程目标

- 形成可复现的比赛提交级 CPU 工程基线
- 保持 RTL、脚本、验证、FPGA 构建与文档口径一致
- 在严格控制回归风险的前提下继续做小步优化

## 3. 设计总览

### 3.1 CPU 核心

- 赛题 ISA 口径：`RV32I` 或 `RV64I`
- 当前冻结性能/提交主口径：`RV32I + Zicsr`
- 当前工程验证能力：`RV32/RV64` 双 XLEN baseline 与 `full-ui` 已有 fresh 结果
- 流水线：`IF / ID / EX / MEM / WB`
- 主要机制：
  - EX/MEM、MEM/WB 前递
  - branch / jump redirect
  - load-use hazard 检测
  - CSR / trap / machine timer interrupt

### 3.2 SoC 组成

- 指令存储：`rtl/YH_rv_sync_imem_rom.v`
- 数据存储：`rtl/YH_rv_dmem_ram.v`
- SoC 顶层：`rtl/YH_rv_cpu_soc.v`
- CPU 顶层：`rtl/YH_rv_cpu.v`
- FPGA 顶层：`fpga/vivado/src/YH_rv_cpu_fpga_top.v`
- 外设：
  - UART
  - timer
  - DONE flag

### 3.3 FPGA 默认内存路径

当前冻结 FPGA 默认配置：

```verilog
parameter integer IMEM_OUTPUT_REG = 0;
parameter integer DMEM_OUTPUT_REG = 0;
```

该默认值已经通过 fresh `impl50` 和 FPGA-like CoreMark probe 验证。

## 6. 当前保留优化

### 6.1 O1：同步 load hazard 收紧

- RTL 位置：`rtl/YH_rv_cpu_hazard_unit.v`
- 当前保留策略：`stall_decode = load_use_hazard`
- 收益：
  - short score `0.912472`
  - strict valid score `0.912465`
  - `riscv-tests rv32/rv64` baseline 无回归

### 6.2 O2：FPGA 默认内存路径收紧

- RTL 位置：`fpga/vivado/src/YH_rv_cpu_fpga_top.v`
- 当前保留默认值：
  - `IMEM_OUTPUT_REG=0`
  - `DMEM_OUTPUT_REG=0`
- 收益：
  - `impl50` fresh 结果通过
  - FPGA-like CoreMark probe 达到 `156442 cycles`，`7.728811 CoreMark/MHz`

### 6.3 明确不保留的方向

- 不能简单放松 `stall_decode` 门控继续做 fetch 前端提分
- 当前已执行的 `request-cursor / pipe-hit / redirect 同拍取指 / FQ-01 ~ FQ-05` 均无可保留收益
- 如需继续前端优化，必须转入 `fetch/request/queue` 解耦设计，并在进入前先收口当前扩展验证
