# YH_rv_cpu 项目进展汇报（2026-04-09 更新版）

> 更新时间：2026-04-09

## 一句话结论

`YH_rv_cpu` 已经收敛出可提交的比赛冻结基线，并补齐了 `2026-04-08` 的 fresh 扩展验证：short CoreMark 为 `0.912472 CoreMark/MHz`，strict CoreMark 为 `0.912465 CoreMark/MHz` 且运行时间 `10.959325s`，`riscv-tests rv32 baseline = 33/33`，`rv64 baseline = 21/21`，`rv32 full-ui = 42/42`，`rv64 full-ui = 54/54`，`impl50 = 2556 LUT / 2170 FF / 4 BRAM / 0 DSP`，`WNS = +5.599ns`。当前本机主工作已转为“初赛提交材料收敛”和“redirect 成本分解后再决定是否继续优化”。

## 当前保留优化

1. `stall_decode = load_use_hazard`
2. FPGA 默认 `IMEM_OUTPUT_REG=0`
3. FPGA 默认 `DMEM_OUTPUT_REG=0`

这三项都已经有 fresh 验证证据，当前继续保留。

## 已完成收口

1. CoreMark 正式短跑口径已经冻结并写入 `doc/coremark_submission_report.md`
2. 已补齐 strict EEMBC `>=10s` 长跑 raw log 与 summary，并保留 `2026-04-08` dated evidence
3. `rv32/rv64 baseline` 与 `rv32/rv64 full-ui` 均已形成 fresh 摘要归档
4. `impl50` bitstream 与 pre-board SOP 已收口，串口口径冻结为 `115200 8N1`
5. `FQ-06A` quick-screen 已完成并拒绝保留，主线 RTL 保持冻结基线
6. `06-汇报材料/初赛提交材料/` 已建立，开始统一维护提交清单、证据索引与答辩口径

## 当前判断

- request-side 微调已经被 `FQ-06A` 与 fresh profile 排除，不适合继续重复投入
- 当前更值得做的是 redirect 成本分解 profiling，再判断是否存在新的非重复优化假设
- 实板闭环仍是外部阻塞，本机内优先级继续低于文档与提交材料收口

## 风险与边界

- FPGA 当前达到的是 board-ready pre-board 状态，不是 board-grade signoff
- `rv32 full-ui` 的 `fence_i` 通过基于扩展 UI 口径 `rv32i_zicsr_zifencei`，冻结比赛 ISA 口径仍是 `RV32I + Zicsr`
- 后续性能优化必须坚持单变量和全矩阵回归，避免破坏当前冻结基线
