## Final Board Claim

After all required evidence is collected, the allowed claim is:

`impl136 exact-CoreMark-ROM fast-gate bitstream is board-programmed and UART-verified on PYNQ-Z2.`

Do not claim strict 10-second board evidence unless a matching >=10 second
board run is separately generated, programmed, captured, and archived.

## Verification Before Claim

After filling the required evidence rows, rerun:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\refresh_impl136_sha256sums.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File .\verify_impl136_evidence.ps1 -RequireBoardEvidence
powershell -NoProfile -ExecutionPolicy Bypass -File .\refresh_impl136_sha256sums.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File .\refresh_impl136_sha256sums.ps1 -CheckOnly
```

The board runner refreshes `SHA256SUMS.txt` before it exits, but this manual
refresh is still required after adding or replacing `board_video_impl136.*`.
The verifier writes its local absolute artifact path to a status file, so the
second refresh is also required after each verifier invocation.

The board claim is allowed only if the verifier reports `VERIFY_RESULT=pass`.
The verifier requires PROGRAM_OK, UART marker status, UART raw log markers, and
`board_video_impl136.[mp4|mov|mkv|avi|webm]`. The PROGRAM status, board-runner
status, UART status, UART raw log, and board video must also be listed in
`SHA256SUMS.txt`.
