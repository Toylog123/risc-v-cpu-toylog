# 集创赛工作区

## 一、项目概况

这个仓库当前只服务一个正式项目：

- 赛题：七星微杯 `RISC-V` 高性能 CPU 设计及 FPGA 验证
- 正式工程：`toylog_cpu`
- 当前目标：先完成初赛可提交的完整闭环，再推进性能优化和展示增强

## 二、当前做到哪一步

当前已经完成：

- 锁定七星微为唯一主赛题
- 建立正式工程 `toylog_cpu`
- 完成 `RV32I` 五级流水第一版，并建立向 `RV32 / RV64` 共线演进的目标
- 打通最小 SoC 闭环：
  - `ROM`
  - `RAM`
  - `UART`
  - `DONE`
  - `timer`
- 打通固件构建链路和 `xsim` 烟测
- 补齐最小机器态 CSR / trap，并通过 trap 烟测
- 打通 machine timer interrupt 烟测
- 整理团队工具链说明和安装清单
- 重构 `01-项目管理`，收拢赛题要求、规划、交接、记录和任务清单

当前还没完成：

- `RV32 / RV64` 共线架构改造
- 更完整的 timer interrupt 与 trap 收口
- `riscv-tests`
- `CoreMark`
- `Vivado` 工程与 FPGA 上板闭环
- 两项性能优化的最终口径冻结

## 五、怎么协作

- 默认同步范围：
  - `toylog_cpu`
  - `04-工具链`
  - `01-项目管理`
- 默认暂存脚本：

```bat
toylog_cpu\scripts\stage_default_sync.bat
```

- 只看默认范围里有哪些改动：

```bat
toylog_cpu\scripts\stage_default_sync.bat --dry-run
```

- 每次修改 `toylog_cpu` 后，同步更新：
  - `toylog_cpu/doc/toylog_cpu_handoff.md`
  - `toylog_cpu/doc/toylog_cpu_change_log.md`
  - `toylog_cpu/doc/toylog_cpu_todo.md`
- 每次项目安排、分工、交接、任务状态发生变化后，同步更新：
  - `01-项目管理/03-过程管理/工作记录.md`
  - `01-项目管理/03-过程管理/工作交接.md`
  - `01-项目管理/03-过程管理/任务清单.md`
