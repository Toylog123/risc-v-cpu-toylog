# 七星微赛题队伍安装清单

这份清单不是“我这台机器装了什么”，而是给队友统一环境用的“按赛题建议整理的最低协作环境”。目标是让大家能够：

- 拉取并阅读 `toylog_cpu`
- 运行 RTL 语法检查和基础仿真
- 编译裸机程序
- 继续做验证、上板和后续开发

## 1. 安装原则

- 以赛题提到的开发流程为准：`RTL -> 测试平台 -> 仿真 -> riscv-tests -> FPGA`
- 以团队协作为准：优先统一“最低可工作环境”，不要每个人都装一整套重工具
- 以当前路线为准：本项目走 `Xilinx FPGA + Vivado + xsim`

## 2. 必须安装

### 2.1 Git

- 用途：同步仓库、提交修改、拉取队友代码
- 要求：命令行可用 `git`

### 2.2 Vivado

- 用途：综合、实现、时序分析、生成比特流、调用 `xsim`
- 建议版本：`Vivado 2025.2`
- 说明：赛题明确提到 `Xilinx Vivado`，这是当前队伍的主线工具

### 2.3 xsim

- 用途：模块级和系统级功能仿真
- 说明：赛题建议使用 `ModelSim`、`VCS`、`Vivado`、`Quartus` 等工具；当前队伍统一采用 `Vivado` 自带的 `xsim`
- 要求：执行 `where xsim` 能找到

### 2.4 Icarus Verilog

- 用途：本地快速语法检查
- 说明：它不是赛题点名工具，但非常适合日常快速检查，能降低仿真前的排错成本
- 安装命令：

```powershell
scoop install iverilog
```

### 2.5 RISC-V 交叉编译器

- 用途：编译 `toylog_cpu` 的裸机程序、导出汇编和二进制
- 当前统一命名：
  - `riscv-none-elf-gcc`
  - `riscv-none-elf-objdump`
  - `riscv-none-elf-objcopy`
- 推荐安装方式：

```powershell
npm install --location=global xpm@latest
xpm install --global @xpack-dev-tools/riscv-none-elf-gcc@15.2.0-1.1
```

## 3. 建议安装

### 3.1 ripgrep

- 用途：快速搜代码、搜文件、查待办标记
- 安装命令：

```powershell
scoop install ripgrep
```

### 3.2 串口工具

- 用途：后续 UART 打印、上板调试
- 说明：名字不强制，能稳定查看串口输出即可

## 4. 可选安装

### 4.1 ModelSim 或 Questa

- 用途：更完整的波形和功能仿真
- 说明：赛题推荐工具之一，但不是当前队伍最低统一环境

### 4.2 VCS

- 用途：大型回归仿真
- 说明：赛题推荐工具之一，但当前阶段没有必要强制统一

### 4.3 SymbiYosys 或 JasperGold

- 用途：形式验证
- 说明：赛题建议可以做形式验证，但不是当前最低门槛

### 4.4 Quartus

- 用途：Intel FPGA 路线
- 说明：赛题提到可用于 FPGA 设计流程，但当前项目不走 Intel 板卡路线

## 5. 当前不需要安装

- `rocket-chip`
  - 原因：不是本项目源码，也不是工具链
- `riscv-gnu-toolchain` 源码树
  - 原因：会让工作区变重；直接装可执行工具链就够了
- 其他大型参考 CPU 仓库
  - 原因：阅读可选，协作开发不依赖

## 6. 赛题建议与本项目映射

- 赛题要求 Verilog/VHDL 设计和测试平台
  - 本项目对应：`toylog_cpu/rtl`、`toylog_cpu/tb`
- 赛题建议做模块级和系统级仿真
  - 本项目对应：`xsim` 为主，`iverilog` 做快速语法检查
- 赛题建议使用 `riscv-gcc`
  - 本项目对应：`riscv-none-elf-gcc`
- 赛题建议运行 `riscv-tests`
  - 本项目对应：后续必须接入
- 赛题要求 FPGA 验证
  - 本项目对应：`Vivado + 开发板`

## 7. 装完后的自检命令

先检查工具是否能找到：

```powershell
where.exe git
where.exe vivado
where.exe xsim
where.exe iverilog
where.exe riscv-none-elf-gcc
where.exe riscv-none-elf-objdump
where.exe riscv-none-elf-objcopy
```

再进入工程执行：

```powershell
cd <你的工作区路径>\toylog_cpu
cmd /c scripts\check_toolchain.bat
cmd /c scripts\check_syntax.bat
cmd /c scripts\build_firmware.bat
```

预期结果：

- `check_toolchain.bat` 能识别 `iverilog`、`xsim`、`Vivado`、`riscv-none-elf-*`
- `check_syntax.bat` 通过
- `build_firmware.bat` 成功生成：
  - `build/sw/toylog_cpu_demo.elf`
  - `build/sw/toylog_cpu_demo.dump`
  - `build/sw/toylog_cpu_demo.bin`

## 8. 统一对外口径

可以直接转发给队友：

> 我们按七星微赛题建议统一一套最低环境。必须装 `Git`、`Vivado 2025.2`、`xsim`、`iverilog`、`riscv-none-elf-gcc`。其中 `Vivado + xsim` 负责 FPGA 和主仿真，`iverilog` 负责快速语法检查，`riscv-none-elf-gcc` 负责编译 `toylog_cpu` 的裸机程序。装完后进入 `toylog_cpu` 跑 `scripts\\check_toolchain.bat`、`scripts\\check_syntax.bat`、`scripts\\build_firmware.bat`，都通过就能同步跟进当前工作。
