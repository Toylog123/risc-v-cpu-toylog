\section{设计目标与方案定位}

\subsection{题目要求分析}

结合官方赛题描述与已公开答疑信息，本项目对题目的理解可以概括为以下几点：
\begin{itemize}
    \item 设计需要在功能正确、性能表现、资源开销和工程可实现性之间取得平衡；
    \item 处理器方案应具备清晰的架构边界，能够支撑技术说明、验证和后续扩展；
    \item 程序存储与数据存储方式可根据实现需要选取，FPGA 原型实现结果可作为阶段性支撑证据；
    \item 在初赛阶段，建立一套能够稳定复现、便于验证且可继续演进的工程基线具有实际意义。
\end{itemize}

考虑到上述要求，本设计采用五级顺序流水结构，以 \texttt{RV32I + Zicsr} 为核心指令子集，
围绕最小 SoC 和 FPGA 原型实现建立完整工程链路。这样的选择有利于在结构复杂度、验证成本和工程可复现性之间取得平衡，
并为后续性能优化、扩展指令支持和实体板验证保留清晰接口。

\subsection{设计目标}

本项目在初赛阶段设置了四项核心目标：
\begin{enumerate}
    \item \textbf{架构清晰}：构建层次分明、职责明确的 CPU 与 SoC 结构，便于说明微架构原理；
    \item \textbf{验证充分}：建立从 smoke、baseline、扩展 ISA 覆盖到 CoreMark 和实现结果的验证矩阵；
    \item \textbf{工程可复现}：使关键脚本、文档、RTL、测试平台和结果摘要保持一致；
    \item \textbf{原型可延伸}：在保持实现约束可控的前提下，为 FPGA 原型和后续性能优化保留清晰路径。
\end{enumerate}
