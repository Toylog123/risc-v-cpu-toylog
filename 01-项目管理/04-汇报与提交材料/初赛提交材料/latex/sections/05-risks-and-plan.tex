\section{结论与后续工作}

\subsection{综合结论}

\texttt{YH\_rv\_cpu} 已经形成较完整的初赛工程形态：架构层面采用边界清晰的五级顺序流水，
验证层面建立了从 smoke、baseline、扩展覆盖到 CoreMark、\texttt{impl50} 和 FPGA-like probe 的证据链，
原型层面完成了可复现的 FPGA 原型适配。综合来看，本设计在初赛阶段具备三项与赛题直接对应的特点：
\begin{itemize}
    \item 五级流水、冒险处理、CSR/trap、最小 SoC 和 FPGA 路径已经形成连续的工程叙事，而不是分散的实验片段；
    \item 文档亮点已经收紧到赛题要求本身，避免把题面未提及的内容包装成主卖点；
    \item 当前剩余缺口也按赛题要求明示，便于继续收口，而不是在正式材料中模糊处理。
\end{itemize}

\subsection{当前边界}

当前说明书中引用的结果已经足以支撑初赛阶段的工程完成度判断，但仍需明确以下边界：
\begin{itemize}
    \item 性能与正式提交主口径仍以 \texttt{RV32I + Zicsr} 为主，扩展 \texttt{full-ui} 结果用于补充说明更广测试范围下的稳定性；
    \item FPGA 结果属于 pre-board closure，而不是实体板连线与最终 I/O delay 约束均已完成的最终 signoff；
    \item 赛题中提到的 IF 预取/预测、简单应用、演示视频、源码打包、连接图和调试日志均仍应视为希望看到的目标项，不能因为当前未完成就从文档叙述中淡化；
    \item 后续若继续开展控制流优化或实现参数调整，仍需在当前验证基线下保持结果和文档同步更新。
\end{itemize}

\subsection{后续工作}

后续工作主要包括：
\begin{enumerate}
    \item 在不依赖实体板的前提下，优先补齐简单应用、演示素材、源码压缩包、Word/PDF 同步和 artifact manifest；
    \item 继续补强源码注释率、模块交互信号说明和提交目录规范，使材料直接对应赛题评分项；
    \item 在保持当前验证基线稳定的前提下，评估是否继续推进控制流前瞻机制、预取或其他能够直接对应题面性能优化条目的增强实现；
    \item 在实体板卡条件具备后，再完成 UART/LED 连线、boot banner 观测、最终板级 I/O 约束确认与长期稳定运行日志。
\end{enumerate}

\subsection{结语}

总体而言，\texttt{YH\_rv\_cpu} 的价值不在于追求极端复杂结构，
而在于把赛题要求、当前实现亮点和后续缺口放到同一张工程图里，使说明书既能展示已完成的工作，也能清楚指出还需要继续收口的部分。
