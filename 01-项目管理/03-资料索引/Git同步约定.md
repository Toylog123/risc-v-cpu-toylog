# Git同步约定

## 默认同步范围

- `toylog_cpu`
- `04-工具链`
- `01-项目管理/02-项目计划`
- `01-项目管理/03-资料索引`

## 这样划分的原因

- `toylog_cpu` 是正式比赛工程，所有 RTL、脚本、文档、交接记录都在这里
- `04-工具链` 直接影响队友是否能复现环境和继续开发
- `01-项目管理/02-项目计划` 保存当前持续协作的计划、任务清单和汇报材料
- `01-项目管理/03-资料索引` 只保留少量会影响协作的索引与说明文档

## 默认不主动同步的目录

- `01-项目管理/01-赛题分析`
- `02-官方与规范`
- `03-参考实现`
- `05-验证测试`

这些目录只有在下面几种情况下再同步：

- 删除大体积无效资料
- 调整目录结构
- 修正文档路径，避免团队继续引用失效内容
- 某项改动已经直接影响队友继续工作

## 默认操作

优先使用：

```bat
toylog_cpu\scripts\stage_default_sync.bat
```

只看这些范围里有哪些改动，可以用：

```bat
toylog_cpu\scripts\stage_default_sync.bat --dry-run
```

## 备注

- 这是一条默认规则，不是死规则
- 如果某次确实涉及其他目录，可以单独补充 `git add` 路径
- Git 仓库本身只保留默认同步范围，其余目录默认是本地资料
