# 七星微赛方答疑整理

> 来源：腾讯文档《第十届集创赛--七星微杯--问题整理》
> 链接：<https://docs.qq.com/sheet/DQ0ZnTG90Z0xmVGFv>
> 更新时间：`2026-03-16`

## 已明确结论

| 主题 | 已确认结论 | 对 `YH_rv_cpu` 的直接影响 |
|------|------------|----------------------------|
| HDL 语言 | `SystemVerilog` 可用 | 当前 RTL 路线可继续 |
| 仿真工具 | `verilator` 可用于快速仿真，但提交时要带源文件和 `testbench` | 当前主线仍以 `xsim` 为主，`iverilog` 做语法检查 |
| 存储实现 | 程序存储器和数据存储器没有强制限制，`BRAM` 可用，不强制 `DDR` | 当前最小 SoC 路线成立 |
| 硬件支持 | 官方不提供服务器和硬件支持 | 开发板、环境和上板流程要自理 |
| 特权架构 | 当前不要求实现完整特权架构指令 | 现阶段保留最小机器态即可 |
| 测试程序 | 没有统一测试指令，`CoreMark` 可适当调整 | 后续以 `riscv-tests + CoreMark` 形成自有闭环 |
| IP 使用 | 部分非核心模块可以用 `Vivado IP` | 外设和辅助模块可以适度借助 IP |

## 对我们当前最重要的落点

- 当前主线应该继续放在基础 CPU、最小 SoC、验证和 FPGA 闭环上。
- 现阶段没有必要为了“完整特权架构”扩题。
- `ModelSim` 不是当前最低环境硬门槛，`Vivado + xsim + iverilog + riscv-none-elf-*` 已足够推进主线。

## 仍待赛方进一步确认

- `Chisel` 是否允许
- 五级流水是固定要求，还是只要不少于五级
- `CoreMark` 指标是否要求 FPGA 实板达标
- `5k LUTs` 是否为硬门槛
- 是否允许引入 `M` 扩展
- 板卡是否统一、是否提供中期检查

## 当前执行建议

1. 不再等待更多口头解释，先按当前已明确要求推进。
2. 文档里把“已明确”和“待确认”分开写，避免误把猜测当结论。
3. 如果后续赛方新增明确答复，再同步更新本文件和 `七星微赛题要求.md`。
