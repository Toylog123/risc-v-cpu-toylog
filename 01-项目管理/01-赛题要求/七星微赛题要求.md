- 当前唯一主赛题已经锁定为七星微杯。

- 当前工程目标不是只停在 `RV32I`，而是从 `RV32I` 稳定基线继续推进到 `RV32 / RV64` 共线。

## 题面目标基线

以下内容仍是赛题主目标，但需结合后续官方答疑理解，不宜机械当作“初赛硬门槛”：

- CPU 基于 `RV32I` 或 `RV64I`
- 基本结构采用五级流水：`IF / ID / EX / MEM / WB`
- 至少完成 `2` 项性能优化
- 必须形成 `RTL -> 仿真 -> FPGA` 完整闭环
- 完整赛程目标包括源代码、测试平台、设计文档、性能/展示材料
- FPGA 原型频率不低于 `50MHz`
- 代码注释率不低于 `30%`

## 当前项目对应口径

- 当前验证基线：`RV32I + Zicsr`
- 当前工程状态：五级流水、最小 SoC、最小机器态 trap、machine timer interrupt 已打通
- 当前 SoC 组成：
  - `ROM`
  - `RAM`
  - `UART`
  - `DONE`
  - `timer`
- 当前烟测状态：
  - `SoC smoke` 通过
  - `trap smoke` 通过
  - `timer irq smoke` 通过

## 2026-04-09 官方答疑补充

- 初赛硬交付更接近“交付方案 / 设计说明书 / 技术文档”。
- 初赛不强制完整性能报告和完整验证报告。
- 初赛测试报告可以只体现一部分核心仿真。
- 初赛不一定非要 FPGA 硬件实现，仿真波形可作为记录。
- 赛方不提供开发板与服务器。
- `SystemVerilog`、`Chisel`、`Verilator` 均允许使用。

这些补充意味着：当前项目的 `CoreMark`、`riscv-tests`、`impl50`、FPGA pre-board 材料属于增强支撑材料，而不是“没有它们就不能交初赛文档”的硬门槛。

## 初赛交付硬要求（按当前答疑）

- 形成完整的设计/技术文档。
- 说明 CPU 架构、SoC 组成、验证方法和 FPGA 适配选择。
- 至少给出核心功能的仿真或验证说明。
- 在文档中诚实写清当前已实现、未实现与后续计划。

## 当前项目建议做到

- CPU 主线稳定运行并可复现
- `UART`、`timer`、基础 smoke 与主要回归可说明清楚
- `CoreMark`、`riscv-tests`、`impl50` 作为增强证据保留
- 将性能优化与资源/复杂度取舍讲清楚

## 当前不做为硬目标

- 乱序执行
- 多发射
- `Linux`
- 复杂 `MMU`
- 依赖大规模软件生态才能验证的高风险特性

## 红线

- 禁止直接复用开源 `RISC-V CPU` 核心 RTL
- 可以参考公开资料理解结构，但不能直接拷核心实现
- 任何新功能都必须先判断是否会拖慢初赛闭环
