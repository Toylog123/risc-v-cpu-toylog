# 七星微赛题要求

> 更新时间：`2026-03-16`
> 当前正式工程：`YH_rv_cpu`

## 当前结论

- 当前唯一主赛题已经锁定为七星微杯。
- 当前项目统一按“先做完整闭环，再做性能优化”的路线推进。
- 当前工程目标不是只停在 `RV32I`，而是从 `RV32I` 稳定基线继续推进到 `RV32 / RV64` 共线。

## 已确认赛程

- 报名截止：`2026-03-31 23:59`
- 初赛提交截止：`2026-05-07`
- 分赛区决赛：`2026-06` 至 `2026-07`
- 全国总决赛：`2026-07` 下旬

## 题面硬要求

以下内容必须作为项目硬基线：

- CPU 基于 `RV32I` 或 `RV64I`
- 基本结构采用五级流水：`IF / ID / EX / MEM / WB`
- 至少完成 `2` 项性能优化
- 必须形成 `RTL -> 仿真 -> FPGA` 完整闭环
- 必须提交源代码、测试平台、设计文档、性能报告和展示材料
- FPGA 原型频率不低于 `50MHz`
- 代码注释率不低于 `30%`

## 当前项目对应口径

- 当前验证基线：`RV32I + Zicsr`
- 当前工程状态：五级流水、最小 SoC、最小机器态 trap、machine timer interrupt 已打通
- 当前 SoC 组成：
  - `ROM`
  - `RAM`
  - `UART`
  - `DONE`
  - `timer`
- 当前烟测状态：
  - `SoC smoke` 通过
  - `trap smoke` 通过
  - `timer irq smoke` 通过

## 初赛必须做到

- CPU 主线稳定运行
- `UART`、`timer` 可用
- `riscv-tests` 基本跑通
- `CoreMark` 数据可复现
- FPGA 上板演示可复现
- 两项性能优化能讲清楚原理、实现和效果

## 当前不做为硬目标

- 乱序执行
- 多发射
- `Linux`
- 复杂 `MMU`
- 依赖大规模软件生态才能验证的高风险特性

## 红线

- 禁止直接复用开源 `RISC-V CPU` 核心 RTL
- 可以参考公开资料理解结构，但不能直接拷核心实现
- 任何新功能都必须先判断是否会拖慢初赛闭环
