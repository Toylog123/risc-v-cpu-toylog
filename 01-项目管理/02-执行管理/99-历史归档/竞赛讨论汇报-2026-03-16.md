## 3. 当前最关键的技术结论

- 当前主线已经从“写一个 CPU 骨架”进入“做一个能交付的比赛工程”阶段。
- 当前最重要的不是再扩散方向，而是三条线同时闭环：
  - `RV64` 语义补齐
  - 验证补齐
  - FPGA 上板补齐
- 赛题明确要求 `RTL -> 仿真 -> FPGA` 完整闭环，所以没有 FPGA 板卡，项目后续会被卡住。
- 当前已经有第一版 FPGA 资源与时序结论：
  - `Slice LUTs = 3445`
  - `Slice Registers = 1962`
  - `BRAM = 0`
  - `DSP = 0`
  - `100MHz` 约束下 `WNS = -2.405ns`
- 这说明当前问题不是“资源不够”，而是“板卡未到位、正式约束未冻结、100MHz 时序还要继续收敛”。

## 4. 当前仍然没完成的关键项

- `CoreMark` 还没有形成稳定可复现跑分记录
- 正式板卡还没到位
- 正式 XDC 还没基于实物板卡冻结
- 上板串口演示还没做
- 两项性能优化需要在文档里冻结成正式口径
- 100MHz 时序还没有收敛
