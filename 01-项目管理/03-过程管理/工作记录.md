## 2026-03-16

### 事项 1：建立正式工程 `toylog_cpu`

- 结论：正式参赛工程从参考实现中独立出来，固定命名为 `toylog_cpu`。
- 结果：后续开发对象清晰，不再和参考目录混在一起。

### 事项 2：完成五级流水第一版

- 结论：当前已经具备 `IF / ID / EX / MEM / WB` 五级流水基线。
- 结果：后续工作重心从“搭骨架”转为“补 SoC、补验证、补上板”。

### 事项 3：整理工具链和协作规则

- 结论：当前团队最低协作环境统一为 `Git`、`Vivado 2025.2`、`xsim`、`iverilog`、`riscv-none-elf-*`、`rg`。
- 结果：队友可以按统一环境接手，不再依赖单机口径。

### 事项 4：清理无关大体积目录

- 结论：删除了 `rocket-chip`、未安装完成的 `riscv-gnu-toolchain` 和本地 RTOS 大包。
- 结果：工作区更轻，后续同步更稳。

### 事项 5：打通最小 SoC 闭环

- 结论：`toylog_cpu` 当前已经能通过最小 SoC 跑固件，串口输出 `toylog_cpu boot`。
- 结果：
  - 新增 `ROM / RAM / UART / DONE / timer`
  - 固件能生成 `.hex`
  - `xsim` SoC 烟测通过

### 事项 6：修复两个关键问题

- 结论：
  - 修复了 `JAL` 后无效指令持续重定向的问题
  - 修复了首次字符串读取拿不到同周期写回值的问题
- 结果：最小 SoC 的 UART 首字节已经恢复正确，烟测稳定通过。

### 事项 7：补齐最小机器态 CSR 与同步 trap

- 结论：`toylog_cpu` 现在已经具备最小可用的 CSR 访问和同步 trap 返回能力。
- 结果：
  - 支持 `mstatus / mtvec / mscratch / mepc / mcause`
  - 支持 `csrrw/csrrs/csrrc`
  - 支持 `ecall / ebreak / mret`
  - trap 烟测通过，输出 `trap ok`

### 事项 8：补齐 machine timer interrupt 最小闭环

- 结论：`toylog_cpu` 当前已经能响应 machine timer interrupt，并完成 `mret` 返回。
- 结果：
  - 支持 `mie / mip / MTIE / MTIP`
  - timer interrupt 烟测通过，输出 `irq ok`
  - 同步异常和定时中断两条 trap 路径都已有独立烟测
