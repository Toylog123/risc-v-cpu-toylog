## 2026-03-17 追加结论

- Vivado 综合口径已经从单一 `100MHz` 扩成双档：
  - `project/reports/clk_10p000ns`
  - `project/reports/clk_20p000ns`
- 最新结果：
  - `100MHz`：`3450 LUT / 1962 FF / 0 BRAM / 0 DSP`，`WNS = -2.487ns`
  - `50MHz`：`3424 LUT / 1962 FF / 0 BRAM / 0 DSP`，`WNS = 7.525ns`
- 当前最适合对老师和队友说的真实口径：
  - 资源不是当前主风险
  - 50MHz 已经有把握
  - 100MHz 还要继续收敛
  - 板卡和正式 `XDC` 仍然是下一阶段关键依赖
- 当天又试了一轮更激进的寄存器使能链优化，但 100MHz 结果更差，已经回退到稳定基线
- 当前关于 BRAM 的真实交接结论：
  - `ROM/RAM` 还没上 BRAM，不是属性没写够
  - 根因是当前 SoC 的取指和 load 仍假设零等待、组合读返回
  - 后续要做 BRAM 化，必须先改 `imem/dmem` 的同步返回语义

## 2026-03-17 当前解读

### 当前项目到了哪一步

- 项目已经从“CPU 骨架搭建”进入“存储结构和 FPGA 口径继续收口”的阶段。
- 当前不是缺主线功能，而是需要把已经跑通的主线继续做扎实：
  - CPU 五级流水稳定
  - 最小 SoC 稳定
  - 基础验证稳定
  - FPGA 综合稳定

### 当前最重要的判断

- 比赛要求的 `50MHz` 口径已经稳定可用。
- `100MHz` 仍是继续优化目标，不是当前交付阻塞点。
- 同步取指已经接入主线，但 `BRAM` 还没有真正被推出来。

### 现在应该怎么向队友或老师解释

- 不是“我们还在搭基础环境”。
- 也不是“已经完全做完 FPGA 优化”。
- 更准确的说法是：
  - 工程主线已经跑通
  - 当前验证链和综合链已经可信
  - 现在正在继续把存储结构往更正式的 FPGA 实现推进

### 当前统一口径

- 最新综合结果以当前测试镜像口径为准：
  - `100MHz`：`4086 LUT / 2040 FF / 1024 LUTRAM / 0 BRAM / 0 DSP`，`WNS = -2.468ns`
  - `50MHz`：`4061 LUT / 2040 FF / 1024 LUTRAM / 0 BRAM / 0 DSP`，`WNS = 7.548ns`
- 当前最真实的结论是：
  - `50MHz` 可以作为比赛演示频率口径
  - `100MHz` 继续优化
  - `BRAM` 仍需后续结构化推进
