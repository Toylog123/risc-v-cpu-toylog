# Git同步约定

## 默认同步范围

- `YH_rv_cpu`
- `04-工具链`
- `01-项目管理`

## 这样划分的原因

- `YH_rv_cpu` 是正式比赛工程，所有 RTL、脚本和工程内技术文档都在这里。
- `04-工具链` 决定队友能不能复现环境并继续开发。
- `01-项目管理` 已经收口到赛题要求、规划、过程管理和资料索引，不再是杂项堆放区。

## 默认不主动同步的目录

- `02-官方与规范`
- `03-参考实现`
- `05-验证测试`

这些目录只在下面情况才同步：

- 调整目录结构
- 修正文档引用路径
- 删除无效大体积资料
- 某次变更已经直接影响团队继续工作

## 默认操作

优先使用：

```bat
YH_rv_cpu\scripts\stage_default_sync.bat
```

只查看默认范围内改动：

```bat
YH_rv_cpu\scripts\stage_default_sync.bat --dry-run
```
