# Git同步约定

## 默认同步范围

- `toylog_cpu`
- `04-工具链`
- `01-项目管理`

## 这样划分的原因

- `toylog_cpu` 是正式比赛工程，所有 RTL、脚本和技术文档都在这里。
- `04-工具链` 决定队友能不能复现环境并继续开发。
- `01-项目管理` 现在已经收口到赛题要求、项目规划、过程管理和资料索引，不再是杂项堆放区。

## 默认不主动同步的目录

- `02-官方与规范`
- `03-参考实现`
- `05-验证测试`

这些目录只在下面几种情况下才同步：

- 删除大体积无效资料
- 调整目录结构
- 修正文档路径，避免继续引用失效内容
- 某项改动已经直接影响团队继续工作

## 默认操作

优先使用：

```bat
toylog_cpu\scripts\stage_default_sync.bat
```

只看默认范围里有哪些改动，可以用：

```bat
toylog_cpu\scripts\stage_default_sync.bat --dry-run
```

## 备注

- 这是一条默认规则，不是死规则。
- 如果某次确实涉及其他目录，可以单独补充 `git add` 路径。
- 当前仓库已经按“正式工程 + 工具链 + 项目管理”收口。
