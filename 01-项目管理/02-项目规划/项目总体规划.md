# 项目总体规划

> 更新时间：2026-03-16  
> 适用赛题：七星微《基于 RISC-V 的高性能 CPU 设计及 FPGA 验证》  
> 当前工程：`toylog_cpu`

## 一、当前状态

- 主赛题已经锁定。
- `toylog_cpu` 已有 `RV32I` 五级流水第一版。
- 工具链基线已经整理完成，当前具备 RTL 语法检查、交叉编译和后续 FPGA 推进基础。
- 当前真正的主线，不再是“继续堆骨架”，而是尽快补齐 SoC、验证和上板闭环。

## 二、总目标

### 1. 初赛目标

- 在 `2026-05-07` 前交出一套完整、可复现、可展示的参赛版本。
- 提交物必须覆盖源码、测试平台、设计文档、性能数据、bitstream、调试记录和视频。
- 初赛以“完整闭环”优先，不以复杂功能堆叠为目标。

### 2. 晋级后目标

- 在初赛闭环稳定的基础上强化性能、展示和答辩质量。
- 优先补强两项性能优化、板级稳定性和数据表达。

## 三、范围控制

### 1. 初赛必须完成

- `RV32I` 五级流水主链路
- `CSR / timer / trap`
- 最小 SoC：`ROM / RAM / UART / timer`
- `riscv-tests`
- `CoreMark`
- FPGA 上板与 `50MHz` 基线
- 两项性能优化
- 代码注释率不低于 `30%`

### 2. 初赛建议完成

- `RV32M`
- 更完整的分支处理
- 轻量级预取或缓存优化
- 基础性能计数器

### 3. 初赛暂不纳入范围

- 乱序执行
- 多发射
- `Linux`
- 复杂 `MMU`
- 需要大量生态适配才能验证的功能

## 六、当前最重要的推进顺序

1. 补齐 `CSR / timer / trap`
2. 建立最小 SoC
3. 接入 `riscv-tests`
4. 接入 `CoreMark`
5. 建立 Vivado 工程和板级约束
6. 输出性能表、设计文档和演示材料

## 七、风险控制

| 风险 | 当前判断 | 控制措施 |
|------|----------|----------|
| 继续堆 RTL 但验证跟不上 | 高 | 优先补验证链，不新增无验证模块 |
| SoC 与板级路径拖晚 | 高 | 先做最小 SoC，不追求复杂外设 |
| FPGA 时序不过 | 中 | 先保稳定运行，再追求更高频率 |
| 文档最后一周集中写 | 高 | 从现在开始同步维护记录和图表 |
| 分工不清导致任务悬空 | 中 | 每项任务只保留一个主负责人 |

## 八、执行原则

- 先闭环，再优化。
- 先可复现，再追求更好看。
- 每个阶段都要留结果，不接受口头完成。
- 任何扩范围动作都必须先判断是否影响初赛交付。
