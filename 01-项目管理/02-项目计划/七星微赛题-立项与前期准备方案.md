## 一、立项结论

- 即日起将 `七星微杯` 作为团队主攻赛题，不再并行推进其他企业命题。
- 队伍准备口径默认按 `B组` 组织；如果最终报名队伍全部为本科生，再调整为 `A组`。
- 前期准备时间窗口按 `2026-03-16` 至 `2026-03-24` 执行，目标是在这一阶段完成立项、资源、工具链和方案冻结。

## 二、为什么选七星微

- 该题同时开放 `A组、B组`，研究生团队可直接参加。
- 题目明确要求 `架构设计 -> RTL 建模 -> 仿真验证 -> FPGA 原型 -> 性能优化` 全链路闭环，适合做成有竞争力的工程作品。
- 赛题与团队当前主线匹配度高：`RISC-V`、`CPU`、`数字电路`、`FPGA`、`性能测评` 都能形成统一主线。
- 题面要求清晰，交付物明确，便于倒排任务和做阶段验收。

## 三、题面硬约束

以下内容必须作为项目基线写入后续所有计划和分工：

- CPU 需基于 `RV32I` 或 `RV64I` 设计。
- 需实现 `五级流水线`：IF / ID / EX / MEM / WB。
- 需至少实现 `2 项性能优化`，可从分支预测、数据旁路、指令预取、Cache 中选择。
- 需完成完整 `Verilog/VHDL -> 仿真 -> FPGA 综合与上板` 流程。
- 需运行 `riscv-tests`、自定义测试程序和应用程序。
- 需提供 `CoreMark` 等性能测试结果。
- FPGA 原型时钟频率要求不低于 `50MHz`。
- 源代码需提交完整 RTL 和 TestBench，且代码注释率不低于 `30%`。
- 禁止直接复用开源 RISC-V CPU 核心代码，如 `Rocket Chip`、`VexRiscv`。
- 最终需提交 Word/PDF 文档、代码包、bitstream、调试日志、性能报告和演示视频。

## 六、技术基线

### 1. 初赛保底方案

- 指令集：`RV32I`
- 架构：`32位、单发射、五级顺序流水`
- 外设：`UART + Timer`，GPIO 如有精力再补
- 存储：先用片上 RAM + 简化总线
- 验证：模块级 TestBench + 系统级 smoke test + `riscv-tests`
- 板级展示：串口输出 + 基础应用程序运行 + CoreMark

### 2. 初赛增强项

- `RV32M`
- 静态或简单动态分支预测
- 数据旁路优化
- 指令预取
- 轻量 I-Cache 或简化 Cache

### 3. 本阶段明确不做

- 乱序执行
- 多发射
- Linux 启动
- 复杂 MMU
- 需要大量生态适配的高风险特性

## 九、关键决策

这些事项必须在 `2026-03-21` 前做出明确结论：

1. 初赛是否只做 `RV32I`，还是一开始就带 `RV32M`。
2. 两项必做性能优化选什么。
3. 是否引入 Cache，还是保守走片上 RAM。
4. 最小 SoC 顶层里保留哪些外设。
5. 板级演示是“串口 + CoreMark”还是再加一个简单应用演示。

## 十、风险与控制

| 风险 | 触发方式 | 控制措施 |
|------|----------|----------|
| 架构贪大 | 一开始就想做 RV64、Cache、预测、RTOS 全部闭环 | 初赛保底固定为 RV32I 五级流水 |
| 工具链不稳 | 仿真工具或综合工具临时不可用 | 2026-03-18 前必须做环境自检 |
| 板卡不匹配 | 开发板资源不足、约束文件缺失 | 2026-03-17 前明确唯一主板卡 |
| 误踩原创性红线 | 直接引用开源 CPU 核 | 只允许参考结构思路，不允许复用核心 RTL |
| 文档滞后 | 后期集中补文档 | 2026-03-22 起同步写架构文档 |

## 十一、当前最重要的结论

- 现在不是再比较赛题的时候，主线已经定为 `七星微`。
- 前期准备阶段的核心任务不是写很多 RTL，而是先把 `组别、资源、工具链、最小架构、两项优化、验证路径` 定下来。
