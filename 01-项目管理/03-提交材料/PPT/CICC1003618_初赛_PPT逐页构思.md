# CICC1003618 初赛 PPT 逐页制作构思

本文档用于后续制作正式初赛 PPT。正式文件建议命名为 `CICC1003618_初赛_PPT.pptx`，并同步导出 `CICC1003618_初赛_PPT.pdf`。文件名中的字段含义为“队伍编号、赛程、文件性质”，不使用加号作为文件名字符。

## 统一风格规范

整体气质：正式技术汇报、学术论文式结构、工程证据导向。画面以黑白灰为主，不使用彩色渐变、装饰性炫光、赛博风背景或多色卡片。所有页面应像同一份正式答辩材料，而不是多套模板拼接。

页面比例：16:9 横版。建议使用白底或极浅灰底，黑色标题，深灰正文，浅灰分隔线。可用一种深灰重点色突出关键数字，但不要使用彩色强调。官方 LOGO 如需出现，优先使用官方原图；若统一黑白灰风格需要，可使用单色版或低饱和处理，但不要自行改造 LOGO 结构。

字体建议：中文使用思源宋体、宋体或等线，英文和数字使用 Times New Roman。标题加粗，正文克制。所有数字等宽或近似等宽排列，保证指标页整齐。

版式系统：每页保留稳定页边距；左上角为章节标签，右下角为页码；标题区域高度固定；正文区域以“一个主图或主表 + 少量注释”为主。避免大段文字、避免三四个大卡片反复堆叠。图表使用细线、浅灰网格、黑色标注。

图形语言：采用线框架构图、流程图、表格、波形截图、指标对比图。线宽统一，箭头简洁。图中文字以模块名、信号名、指标名为主，少写完整句子。

提交注意：PPT 中不得出现学校、学院、老师、导师、个人身份等信息。可出现队伍编号 `CICC1003618` 与团队名称“芯银河战队”。AI 工具使用声明需要单独成页。

## 第 1 页：封面

页面目标：第一眼说明作品名称、赛题方向、队伍编号和团队名称，风格接近大赛正式报告封面。

版式：白底居中封面。顶部放大赛 LOGO，居中偏上；中部放标题；下方用细横线表单式列出报告类型、参赛杯赛、作品名称、队伍编号、团队名称。底部留白，不放学校和人员信息。

页面文字：标题为“基于 RISC-V 的五级流水 CPU 设计与 FPGA 验证”。报告类型写“初赛作品汇报”。参赛杯赛写“第十届全国大学生集成电路创新创业大赛”。作品名称写“基于 RISC-V 的五级流水 CPU 设计与 FPGA 验证”。队伍编号写“CICC1003618”。团队名称写“芯银河战队”。

绘制提示：生成一张正式学术报告封面风格的 PPT 页面，黑白灰配色，白底，官方 LOGO 居中置顶，标题使用较大黑色宋体，表单信息使用细横线分隔，版面干净庄重，无学校和人员身份信息。

## 第 2 页：快速预览 1 - 关键指标

页面目标：在评委翻到正文前，快速看到最重要的可验证结果。

版式：左侧为一句定位语和一条竖向灰色基准线，右侧为指标矩阵。指标不要做彩色卡片，采用开放式大数字排版，每个数字下面配一行指标名。底部放一条证据链说明。

页面文字：定位语为“面向 PYNQ-Z2 的轻量级高性能 RISC-V 软核 CPU”。核心指标依次为 `4.137461 CoreMark/MHz`、`2.908287 DMIPS/MHz`、`4934 LUT`、`50.0 MHz`、`PROGRAM_OK`。底部证据说明为“仿真、性能基准、Vivado 实现、PYNQ-Z2 上板下载形成闭环证据”。

视觉元素：五个指标用统一字体和对齐方式呈现；`4934 LUT` 旁边标注“LUT < 5000 约束内”；`PROGRAM_OK` 旁边标注“xc7z020_1”。

绘制提示：生成黑白灰技术指标总览页，右侧五个大数字开放排版，不要卡片化，不要彩色；左侧用一句话概括作品定位；底部用细线串联“仿真、性能、资源、时序、上板”证据链。

## 第 3 页：快速预览 2 - 核心亮点

页面目标：把“为什么这个设计值得看”讲清楚，突出赛题要求中已经实现的内容。

版式：中间放一个简化 CPU 核心线框，四周用四个方向标注亮点。四个方向分别是 ISA 热点增强、流水控制优化、低资源低功耗取向、工程验证闭环。

页面文字：ISA 热点增强写“Zmmul、Zba/Zbb/Zbs 已进入提交路径”。流水控制优化写“JAL early redirect、forwarding、stall/flush 协同”。低资源低功耗写“顺序五级流水、参数化扩展、避免高成本除法器”。工程验证闭环写“RTL、TestBench、CoreMark、Dhrystone、Vivado、PYNQ-Z2 日志”。底部小字写“探索路径与冻结提交路径分开呈现，PPT 只展示已实证能力”。

视觉元素：四个方向用细线从 CPU 核心发散，线条深灰；每个亮点最多三行，避免段落。

绘制提示：生成一页黑白灰核心亮点图，中间是简化五级流水 CPU 方框，四周四组亮点以细线连接；强调 Zmmul、Zba/Zbb/Zbs、JAL early redirect、低资源、验证闭环；不要使用彩色图标或装饰背景。

## 第 4 页：赛题理解与设计目标

页面目标：说明设计目标来自赛题约束，而不是单纯追分。

版式：左侧列赛题约束，右侧列设计选择，中间用箭头连接。底部给一句设计原则。

页面文字：赛题约束包括“支持 RV32I/RV64I 指令集之一”“FPGA 原型系统稳定运行”“主频不低于 50 MHz”“提交仿真与 FPGA 测试数据”“代码注释率不低于 30%”“资源尤其是 LUT 受限”。设计选择包括“RV32I 五级顺序流水”“PYNQ-Z2 适配”“50 MHz CPU 时钟”“CoreMark/Dhrystone 双基准”“参数化 ISA 扩展”“LUT < 5000 内冻结提交”。底部原则为“先保证可综合、可验证、可上板，再对热点路径做选择性增强”。

视觉元素：两列对照不使用彩色勾叉，用黑色实线和灰色层级表示对应关系。

绘制提示：生成一页赛题约束到设计选择的对照流程图，左列为要求，右列为实现策略，中间箭头连接，白底黑字灰线，结构像论文中的方法论图。

## 第 5 页：总体架构

页面目标：展示 CPU、片上存储、MMIO、UART、LED、FPGA 顶层之间的关系。

版式：整页以系统架构图为主。上方标题，下方小注释。架构图从左到右为 JTAG/bitstream、PYNQ-Z2 FPGA Top、SoC、CPU Core、ROM/RAM/MMIO、UART/LED。

页面文字：图中模块名包括 `YH_rv_cpu_fpga_top`、`YH_rv_cpu_soc`、`YH_rv_cpu`、`IF/ID/EX/MEM/WB`、`ROM`、`RAM`、`MMIO`、`UART TX`、`LED`。页脚注释为“FPGA 顶层负责时钟、复位、Pmod UART 与 LED 适配；SoC 层负责片上存储与 MMIO 映射；CPU Core 负责流水线执行与控制”。

视觉元素：所有模块用白底黑边矩形；存储和 MMIO 用浅灰填充；箭头表示指令流、数据流和外设输出。

绘制提示：生成一页黑白灰 SoC 总体架构图，模块边框细线，重点展示 CPU Core、五级流水、ROM/RAM/MMIO、UART/LED 和 PYNQ-Z2 顶层连接关系，不要使用三维芯片效果。

## 第 6 页：五级流水与控制通路

页面目标：说明五级流水的核心控制机制和正确性保障。

版式：横向五段流水图占页面主体，下面放三条控制线：redirect、stall、flush/forwarding。右侧放一个小型说明区，解释“提前判断 + 后级校验”的原则。

页面文字：五级流水为 `IF`、`ID`、`EX`、`MEM`、`WB`。控制信号为 `redirect`、`stall`、`flush`、`forwarding`。右侧说明写“ID 阶段进行轻量分支/跳转提前处理；EX 阶段保留校验和兜底重定向；hazard unit 统一处理停顿、刷新与前递”。

视觉元素：五级流水用等宽线框；控制线用不同灰度和线型，不用彩色；重定向箭头从 ID/EX 指向 IF。

绘制提示：生成一页五级流水控制通路图，黑白灰，五个阶段横向排列，重点标注 redirect、stall、flush、forwarding；右侧用极简文字说明提前判断和后级校验。

## 第 7 页：ISA 扩展与热点优化

页面目标：把已实现并有证据的 ISA 扩展写成亮点，突出与性能热点的关系。

版式：使用一张四列表格，列名为“扩展/机制、优化对象、收益方向、验证证据”。表格风格像论文表格，细线，浅灰表头。

页面文字：`Zmmul` 行写“乘法热点；减少软件乘法序列；Zmmul diagnostic PASS”。`Zba/Zbb/Zbs` 行写“地址生成、位操作、位提取；降低热点指令序列长度；Bitmanip subset 回归”。`JAL early redirect` 行写“跳转重定向；减少前端空转；JAL fast diagnostic”。`参数化探索路径` 行写“Zbc/XThead/IDBR；用于性能空间探索；最终提交路径按 LUT 和时序约束筛选”。

视觉元素：表格不要过密，每行保留足够行高；已实证项用黑色实心小点，探索项用空心小点。

绘制提示：生成一页论文式技术表格，黑白灰配色，展示 Zmmul、Zba/Zbb/Zbs、JAL early redirect 和参数化探索路径；区分已进入提交路径与探索路径，不要夸大未冻结能力。

## 第 8 页：低功耗与低资源设计取向

页面目标：解释为什么采用克制的微架构选择，体现 PPA 平衡。

版式：左侧是“没有采用的高成本方案”，右侧是“采用的轻量级方案”，中间用天平或 PPA 平衡示意连接。底部放资源结论。

页面文字：左侧列“复杂动态预测、乱序执行、完整除法器、过宽旁路网络”。右侧列“顺序五级流水、热点乘法/位操作增强、低成本静态控制、参数化功能开关”。底部结论写“性能提升集中在热点路径，资源和时序保留余量”。

视觉元素：不要把“未采用方案”画成失败或红色叉号，用浅灰表示高成本路径；采用路径用黑色线框强调。

绘制提示：生成一页黑白灰 PPA 平衡图，左侧高成本方案灰化，右侧轻量化方案突出，中间用简单天平或对称结构表示性能、面积、功耗的平衡。

## 第 9 页：RTL 建模规范与工程组织

页面目标：展示代码结构完整、边界清楚、可复现。

版式：页面主体为目录树和模块职责映射。左侧目录树，右侧对应职责。下方放注释率和复现脚本两个证据点。

页面文字：目录树包括 `rtl/`、`tb/`、`sw/`、`fpga/`、`scripts/`。职责说明为“RTL：CPU 与 SoC 设计代码；TB：模块与系统级 TestBench；SW：RISC-V 测试程序、CoreMark/Dhrystone 端口；FPGA：PYNQ-Z2 约束与 Vivado 脚本；scripts：构建、仿真、统计和日志整理”。证据点写“源码注释率保守口径 31.87%”“语法检查脚本退出码 0”。

视觉元素：目录树用等宽字体；右侧职责用短句，不做长段。

绘制提示：生成一页工程目录与职责映射图，黑白灰，左侧代码目录树，右侧职责说明，底部突出注释率 31.87% 和语法检查通过。

## 第 10 页：验证体系

页面目标：说明验证不是单一 benchmark，而是分层闭环。

版式：用自下而上的四层金字塔或四层流水验证链。四层为“指令/模块定向测试、SoC smoke、性能基准、FPGA 实现与下载”。

页面文字：第一层写“Zmmul、Bitmanip、分支、Load-use、Trap、Timer”。第二层写“SoC smoke、存储和 MMIO 通路”。第三层写“CoreMark、Dhrystone、优化前后对比”。第四层写“Vivado 资源时序、bitstream、PYNQ-Z2 PROGRAM_OK”。旁注写“覆盖功能正确性、性能数据、资源时序、上板证据”。

视觉元素：每层使用浅灰分隔，最终上板层用黑色边框强调。

绘制提示：生成一页分层验证体系图，四层从仿真到上板，黑白灰，重点突出从指令正确性到 FPGA 下载日志的闭环。

## 第 11 页：关键波形与时序证据

页面目标：用波形证明关键控制机制可观察、可解释。

版式：放 2 到 3 张关键波形截图，每张下方放两条注释。波形截图保持黑白灰或低饱和，信号名清晰。

页面文字：波形 1 标题“ID 分支快速重定向”，注释“Fetch Redirect 拉高，EX Redirect 保持低；前端提前修正 PC”。波形 2 标题“JAL 快速重定向”，注释“跳转目标提前进入取指路径；减少无效取指”。波形 3 标题“Load-use 快速路径”，注释“Stall Decode 在该场景保持低；前递/等待逻辑未引入额外停顿”。页脚写“波形来自性能与验证报告的波形与时序证据目录”。

视觉元素：波形不要放太多信号，每张保留重点信号；用黑色箭头或括号标注观察点。

绘制提示：生成一页关键波形证据页，三张横向波形小图，黑白灰，分别标注 ID 分支、JAL、Load-use 的观察点；每张图只放两条短注释。

## 第 12 页：性能结果

页面目标：集中展示 CoreMark 与 Dhrystone 结果，并说明数据独立来源。

版式：左侧为 CoreMark 指标，右侧为 Dhrystone 指标，中间用细竖线分隔。底部放测试条件。

页面文字：CoreMark 区域写 `4.137461 CoreMark/MHz`、`2416941 ticks`、`413.746136 iter/s`。Dhrystone 区域写 `2.908287 DMIPS/MHz`、`510986 Dhrystones/s`、`10 runs`。底部测试条件写“CPU 50.0 MHz；CoreMark 与 Dhrystone 分别独立运行；结果以本地仿真、性能日志和验证报告为准”。

视觉元素：两个大数字是页面第一视觉；其他数字较小；不要使用仪表盘卡片。

绘制提示：生成一页双指标性能结果页，左 CoreMark 右 Dhrystone，黑白灰，大数字开放排版，底部写测试条件和证据来源。

## 第 13 页：优化前后对比

页面目标：用对比证明优化有效，同时保持对资源约束的解释。

版式：上半部分为 CoreMark 对比条形图，下半部分为 DMIPS 对比条形图，右侧放“资源约束说明”。条形图用灰度深浅，不用彩色。
