# 分赛区决赛技术说明书更新要点

更新时间：2026-05-08

本文件用于后续重写分赛区技术说明书。当前正式 PDF 暂沿用初赛版本；重写时应把下面内容融合进正文，而不是作为“版本迭代记录”直接堆进去。

## 一、建议新增或强化的核心叙事

技术说明书应从“一个可上板的 RV32I 五级流水 CPU”升级为“一个面向 FPGA 软核场景、可参数化裁剪、在性能/面积/功耗之间可选择工作点的 RISC-V CPU 原型”。重点不再只是达到初赛指标，而是说明如何通过架构参数、热点扩展和验证闭环形成可持续优化能力。

## 二、建议写入快速预览页的亮点

| 亮点 | 建议表述 |
|---|---|
| 性能 | post-初赛最高探索结果达到 7.266613 CoreMark/MHz 与 10.163426 DMIPS/MHz |
| 低功耗 | 低功耗 no-IDBR 候选达到 7.208501 CoreMark/MHz、10.154360 DMIPS/MHz，默认活动功耗估计 0.239 W |
| 面积 | 面积 no-Zbc 候选降至 5097 LUT，保持 50 MHz 时序收敛 |
| 参数化 | 支持 XLEN、Zmmul、Bitmanip、Zbc、XThead、IDBR 等开关，便于按赛题目标选择 PPA 工作点 |
| 上板 | PYNQ-Z2 / `xc7z020clg400-1`，50 MHz，bitstream、Vivado 报告、UART 演示证据可追溯 |
| 验证 | 覆盖 RTL/TestBench、指令级测试、CoreMark/Dhrystone、资源时序、功耗估计、FPGA 上板 |

## 三、架构章节建议补充

1. 参数化设计  
   写清楚当前 CPU 采用五级顺序流水，核心参数包含 XLEN、扩展开关、低功耗裁剪开关。PYNQ-Z2 主验证路径采用 RV32I 方向，RV64 相关路径作为参数化验证与扩展基础保留。不要把 RV64 写成当前主提交 bitstream 的同等上板能力。

2. 热点扩展选择  
   将 Zmmul、Zba/Zbb/Zbs、Zbc、XThead、IDBR 解释为围绕 CoreMark/Dhrystone 热点逐步筛选出来的候选集。强调不是简单堆功能，而是根据资源、时序、功耗进行启停对比。

3. 控制流优化  
   介绍 JAL early redirect、轻量分支路径和可选 ID branch EX-forward。重写时应强调：IDBR 在最高性能路径中有价值，但低功耗候选关闭后仍满足性能目标，并带来更好的功耗和时序余量。

4. 低功耗设计  
   增加“参数化裁剪降低无效切换”的小节。低功耗候选通过关闭 ID branch EX-forward，把默认活动功耗估计从 0.276 W 降到 0.239 W，动态功耗从 0.168 W 降到 0.131 W。

5. 面积设计  
   增加“no-Zbc 面积候选”说明：当 CRC 快路径由软件/源码优化覆盖后，Zbc 硬件对当前分数不再是必要条件，关闭 Zbc 后 LUT 降到 5097。该候选功耗不是最低，因此定位为面积优先工作点。

## 五、正式写作时的口径边界

- CoreMark7 结果需要标明来自 post-初赛探索节点，当前日志为短仿真 host-parsed 结果，严格 EEMBC 10 秒规则仍需后续长运行验证。
- Dhrystone 10+ DMIPS/MHz 来自 20-run 主路径，100-run 因 RV32 `long` 乘积溢出不作为评分数据。
- Benchmark 源码补丁必须在源码说明或验证报告中披露，不要把所有提升都写成 RTL 硬件提升。
- 初赛 PDF 可以作为基线沿用，但分赛区正式稿需要删除“初赛当前版本”等自我管理语气，改成面向评委的正式汇报口吻。
