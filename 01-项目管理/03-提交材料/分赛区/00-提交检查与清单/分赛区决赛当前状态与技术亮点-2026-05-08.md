## 二、当前可写入分赛区材料的技术亮点

1. 参数化 RISC-V CPU 架构  
   CPU 以五级顺序流水为主体，支持 XLEN、乘法扩展、Bitmanip 子集、Zbc、XThead 扩展、ID 阶段分支前递等参数化开关。当前 PYNQ-Z2 与性能验证主路径采用 RV32I 方向；RV64 相关路径作为参数化验证能力保留，正式表述应避免写成“RV32/RV64 同时上板提交”。

2. 面向 FPGA 的 PPA 平衡设计  
   设计没有堆叠乱序、复杂动态预测或完整除法器，而是在热点路径上选择低面积收益高的增强项。最终探索节点在 50 MHz 下保持时序收敛，并给出低功耗候选与面积候选两条路线。

3. 热点 ISA 与执行路径优化  
   通过 Zmmul、Zba/Zbb/Zbs、XThead 条件移动/访存索引等可选扩展覆盖 CoreMark 与 Dhrystone 的整数热点。Zbc 在后续探索中证明并非当前低功耗最优必需项，因此分赛区可强调“按 benchmark 热点与 FPGA 资源约束进行扩展裁剪”。

4. 前端控制流优化  
   JAL 早期重定向、轻量级分支处理、可选 ID branch EX-forward 形成了前端控制流优化组合。低功耗候选中关闭 ID branch EX-forward 后，CoreMark 仍保持 7.208501 CoreMark/MHz，同时 LUT 和功耗下降，体现了参数化裁剪价值。

5. CoreMark7 / DMIPS10 性能提升  
   post-初赛探索节点达到 7.266613 CoreMark/MHz 与 10.163426 DMIPS/MHz。低功耗推荐候选达到 7.208501 CoreMark/MHz 与 10.154360 DMIPS/MHz。与 CoreMark6/DMIPS4.7 节点相比，最高 CoreMark 提升约 17.50%，DMIPS 提升约 116.07%。

6. 低功耗候选  
   低功耗 no-IDBR bitstream：5601 LUT、2317 FF、4 BRAM、15 DSP，WNS +0.864 ns、WHS +0.111 ns，默认活动功耗估计 0.239 W，其中动态功耗 0.131 W。该候选是当前推荐写入分赛区材料的低功耗结果。

7. 面积裁剪候选  
   no-Zbc + XThead + no-IDBR bitstream：5097 LUT、2316 FF、4 BRAM、15 DSP，WNS +0.546 ns、WHS +0.118 ns。功耗估计高于低功耗候选，因此适合表述为“面积优先候选”，而不是低功耗推荐候选。

8. 验证闭环完整  
   现有材料覆盖 RTL/TestBench、指令与模块定向测试、CoreMark/Dhrystone 性能日志、Vivado 资源时序、PYNQ-Z2 bitstream、UART 输出与演示视频。分赛区材料应继续强化“仿真、实现、上板、性能、功耗、可复现”的证据链。

9. 上板与应用演示基础已具备  
   初赛阶段已经完成 PYNQ-Z2 下载、UART 连接、串口输出和应用演示视频。分赛区后续应在最新 CoreMark7 低功耗 bit 上复验串口输出，把性能指标、tick、PC、状态流做成更适合录制的视频证据。

10. AI 辅助工程流  
   OpenAI 可继续表述为工程脚本梳理、性能热点定位、实验日志核对、验证流程组织；DeepSeek 可继续表述为文档润色与表达压缩。正式文档中仍需保留 AI 使用声明，并强调指标来自本地仿真、Vivado 报告和上板证据。

## 四、注意事项

- CoreMark7 当前仍是短仿真 host-parsed 结果，日志中 `strict_eembc_10s_compliant=no`。分赛区正式表述应说明这是可复现短运行性能验证，若赛方要求严格 EEMBC 10 秒规则，需要再扩展运行时间或给出说明。
- Dhrystone 100-run 曾因 RV32 `long` 的 `HZ * Number_Of_Runs` 溢出被排除，当前采用 20-run 数据。正式报告应保留这一解释，避免评审追问时显得口径不一致。
- 低功耗与面积是两条不同候选：低功耗推荐 no-IDBR 版本，面积推荐 no-Zbc 版本。不要把 5097 LUT 与 0.239 W 写成同一个 bit 的结果。
