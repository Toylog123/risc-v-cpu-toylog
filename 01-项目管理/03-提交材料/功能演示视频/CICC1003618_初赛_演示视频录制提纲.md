# CICC1003618 初赛演示视频录制提纲

本目录仅保留录制提纲，正式视频由人工录制与剪辑完成。视频建议横屏 `1920x1080`，时长约 5 分钟，不出现参赛单位和人员身份信息。

## 0:00-0:25 开场与作品定位

画面展示封面页或技术说明书封面。口播：作品为“基于 RISC-V 的五级流水 CPU 设计与 FPGA 验证”，面向七星微企业命题，队伍编号 `CICC1003618`，团队名称“芯银河战队”。说明目标是在 PYNQ-Z2 上实现可综合、可仿真、可下载的轻量级 RISC-V CPU。

## 0:25-1:10 架构讲解

画面切到系统总体架构图和五级流水图。口播 CPU Core、最小 SoC、ROM/RAM、MMIO、UART、FPGA 顶层之间的关系。强调 IF、ID、EX、MEM、WB 五级流水，redirect、stall、flush 和 forwarding 构成主要控制通路。

## 1:10-2:00 核心优化讲解

画面展示 ISA 与优化表。依次讲最终冻结路径 `Zmmul`、`Zba/Zbb/Zbs`、`JAL early redirect`；再补充说明 `Zbc/XThead/IDBR` 为参数化探索路径，未进入本次 PYNQ-Z2 冻结 bitstream。明确说明自动更新 memidx 和 C/RVC 不作为已实现能力宣称。讲稿重点是“在 LUT < 5000 下选择低成本高收益优化，而不是堆复杂结构”。

## 2:00-2:55 功能验证演示

画面展示终端或报告中的回归结果：SoC smoke、Zmmul、Bitmanip 最终子集、Branch predict、CoreMark、Dhrystone，以及 Zbc/XThead/IDBR 探索路径回归记录。再展示 2 到 3 张关键波形：ID 分支快速重定向、JAL 快速重定向、load-use 快速路径。口播每张波形的判断依据，例如 Fetch Redirect 与 EX Redirect 的关系、Stall Decode 是否拉高。

## 2:55-3:50 性能验证演示

画面展示性能表和原始日志目录。口播正式报告值：`4.137461 CoreMark/MHz`、`2.908287 DMIPS/MHz`。说明 CoreMark 由 raw ticks 与固定时钟解析，CRC 保留在日志中；Dhrystone 独立运行，不由 CoreMark 外推，优化构建使用 `-O3 -flto -fwhole-program` 且仅处理生成构建副本。展示优化前后对比，说明提交路径在性能、LUT 和时序之间取得平衡；高性能探索值 `5.162186 CoreMark/MHz` 因资源/时序不满足冻结条件，仅作为后续优化参考。

## 3:50-4:35 FPGA 上板证据

画面展示 PYNQ-Z2 板卡、Vivado Hardware Manager 或下载日志截图、资源和时序报告。必须出现 `PROGRAM_OK: xc7z020_1`、`4934 LUT`、`WNS +0.440 ns`、`50.0 MHz`。如果能外接 USB-UART，展示 Pmod B 连接和串口打印；如果没有外接 USB-UART，就口播说明 PYNQ-Z2 Micro-USB 串口连接 PS 侧，PL 软核 UART 已约束到 Pmod B，需要外接 `3.3 V` USB-UART 才能采集文本日志。

画面回到快速预览页。口播总结：作品完成 CPU 架构设计、RTL 建模、指令与性能验证、PYNQ-Z2 FPGA 适配和上板下载证据；核心特点是高性能、低资源、可复现、证据链完整。最后停留在关键指标页 2 秒。

## 串口录制补充

板载 Micro-USB 在 Windows 中识别为 `USB Serial Port (COM8)`，本轮以 `115200 8N1` 打开并发送回车，采集结果为 `0` 字符。该现象符合 PYNQ-Z2 的连接边界：板载 Micro-USB 串口连接 Zynq PS 侧，不直接连接 PL 软核 UART。

正式视频若要展示软核文本输出，需要外接 `3.3 V` USB-UART 到 Pmod B。连接方式为：外部适配器 `RX` 接 `JB1 / Y14 / uart_rxd_out`，外部适配器 `TX` 可选接 `JB0 / W14 / uart_txd_in`，`GND` 共地。采集命令见 `FPGA原型系统/fpga_artifacts_pynq_z2/串口录制说明-2026-05-05.md`，推荐录到 `logs/pl_uart_capture_YYYYMMDD_HHMMSS.txt` 后在视频中展示。

录制时口播不要说“板载 COM8 就是软核串口”。应说明：板载 Micro-USB 用于 JTAG 下载和 PS 侧串口识别；YH_rv_cpu 位于 PL 侧，软核 UART 已在约束中引出到 Pmod B，需要外接 3.3 V USB-UART 才能采集 `YH_rv_cpu boot` 或基准测试打印。
