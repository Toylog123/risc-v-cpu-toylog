# PYNQ-Z2 串口录制说明

更新时间：`2026-05-05`

## 连接结论

PYNQ-Z2 板上的 Micro-USB PROG/UART 口会在 Windows 中枚举为 FTDI USB Serial Port。本机复核到的端口为：

| 项目 | 结果 |
|---|---|
| Windows 端口 | `USB Serial Port (COM8)` |
| 设备 ID | `FTDIBUS\VID_0403+PID_6010+1234-TULB\0000` |
| 复核设置 | `115200 8N1, no parity, no flow control` |
| 采集结果 | `serial_probe_COM8_20260505_092755.txt` 中 `captured_chars=0` |

该 Micro-USB 串口连接 Zynq PS 侧，不直接连接 PL 软核 UART。因此，当前 YH_rv_cpu 软核的 UART 文本输出不能从板载 `COM8` 直接采集。

## PL 软核 UART 录制连接

YH_rv_cpu FPGA 顶层已经实例化 `YH_rv_uart_tx`，demo 固件会通过 MMIO 地址 `0x1000_0000` 输出 `YH_rv_cpu boot\n`。在 PYNQ-Z2 提交约束中，PL 软核 UART 接到 Pmod B：

| 信号 | PYNQ-Z2 引脚 | 连接方式 |
|---|---|---|
| `uart_rxd_out` | `JB1 / Y14` | FPGA TX，连接外部 USB-UART 适配器 RX |
| `uart_txd_in` | `JB0 / W14` | FPGA RX，可选，连接外部 USB-UART 适配器 TX |
| `GND` | Pmod B GND | 与外部 USB-UART 适配器共地 |
| 电平 | `3.3 V` | 只使用 3.3 V USB-UART，禁止 5 V 串口电平 |

录制视频时建议同时保留两条线：

1. PYNQ-Z2 Micro-USB 连接电脑，用于 Vivado Hardware Manager 下载 bitstream。
2. 外部 3.3 V USB-UART 连接 Pmod B，用于采集 PL 软核文本输出。

## 采集命令

在源码目录执行：

```powershell
cd D:\BaiduSyncdisk\02_icdc_workspace\01-项目管理\03-提交材料\源代码\CICC1003618_初赛_源代码
powershell -ExecutionPolicy Bypass -File .\scripts\capture_uart.ps1 -List
powershell -ExecutionPolicy Bypass -File .\scripts\capture_uart.ps1 -Port COMx -Seconds 20 -LogPath ..\..\FPGA原型系统\fpga_artifacts_pynq_z2\logs\pl_uart_capture_20260505.txt
```

其中 `COMx` 应替换为外部 USB-UART 适配器对应的端口，不是板载 PS 侧 `COM8`。

## 录制时的预期现象

1. 使用 Vivado 下载冻结 bitstream，Hardware Manager 日志出现 `PROGRAM_OK: xc7z020_1`。
2. 打开外部 USB-UART 对应 COM 口，参数为 `115200 8N1`。
3. 将 `SW0` 拨到释放复位状态，或重新按连接流程复位软核。
4. demo 固件路径下，串口应出现 `YH_rv_cpu boot`。
5. 若烧录 CoreMark/Dhrystone 对应 ROM，则串口会输出基准测试文本；最终成绩仍以性能与验证报告中的冻结日志为准。

## 视频口播建议

可简洁说明：PYNQ-Z2 板载 Micro-USB 同时承担 JTAG 下载和 PS 侧串口识别；本作品 CPU 位于 PL 侧，因此软核 UART 通过 Pmod B 引出，并使用外接 3.3 V USB-UART 采集。这样可以避免把 PS 控制台误认为软核输出，也能保证上板日志来源清晰、可复现。
